# Документация по архитектуре

Дата: 20.02.2026  
Проект: Flowwow

## 1. Общая архитектура

Система построена по клиент‑серверной схеме:
- **Frontend**: статические HTML/CSS/JS‑страницы, отдаваемые backend‑ом.
- **Backend**: REST API на Spring Boot.
- **База данных**: реляционная БД (JPA/Hibernate).

Основные слои backend:
- **Controller**: REST‑контроллеры API.
- **Service**: бизнес‑логика.
- **Repository**: доступ к данным (Spring Data JPA).
- **Entity/DTO**: модели сущностей и транспортные объекты.

## 2. Модули системы

1. **Модуль авторизации**
- Назначение: регистрация, вход, выдача JWT.
- Входные данные: email, пароль.
- Результат: токен доступа, профиль пользователя.

2. **Модуль работы с БД**
- Назначение: CRUD для товаров, категорий, изображений, заказов, отзывов.
- Входные данные: DTO + параметры запросов.
- Результат: записи в БД и выборки для UI.

3. **Модуль админ‑панели**
- Назначение: управление товарами/категориями/заказами/отзывами.
- Входные данные: формы админ‑UI.
- Результат: изменения в БД и отображение в панели.

4. **Модуль поиска и фильтрации**
- Назначение: поиск товаров по запросу, фильтрация по параметрам.
- Входные данные: строка поиска и фильтры.
- Результат: отфильтрованный список товаров.

5. **Пользовательский интерфейс**
- Назначение: каталог, карточки товара, корзина, избранное, оформление заказа.
- Входные данные: действия пользователя в UI.
- Результат: навигация, отображение данных, отправка API‑запросов.

## 3. Потоки данных (упрощённо)

1. Пользователь открывает страницу → backend отдаёт HTML/JS/CSS.  
2. UI запрашивает данные через REST API.  
3. Backend обращается к БД через Repository.  
4. Ответ возвращается в UI и отображается.

## 4. Инфраструктура сборки фронтенда

- Сборка JS/CSS через `esbuild`.
- Единый бандл JS на страницу.
- Единый минифицированный CSS.
