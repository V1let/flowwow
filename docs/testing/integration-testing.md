# Интеграционное тестирование программного продукта

Дата: 20.02.2026  
Проект: Flowwow  
Формат: протокол интеграционных испытаний + описание взаимодействий модулей

## 1. Описание взаимодействия модулей

### 1.1 Frontend ↔ Backend
- Frontend отправляет запросы в API (`/api/*`), получает JSON.
- Backend обрабатывает авторизацию, бизнес‑логику, возвращает данные для UI.

### 1.2 Backend ↔ База данных
- Backend использует JPA/Hibernate для доступа к БД (CRUD для сущностей).
- Целостность обеспечивается внешними ключами и ограничениями.

### 1.3 Админ‑панель ↔ База данных
- Админка использует защищённые API (`/api/admin/*`) для управления сущностями.
- Все действия (создание/редактирование/удаление) отражаются в БД.

### 1.4 Поиск и фильтрация
- Frontend отправляет параметры фильтров (`categoryId`, `minPrice`, `maxPrice`, `search`).
- Backend выполняет поиск и возвращает страницу товаров.

## 2. Сценарии проверки

### 2.1 Frontend ↔ Backend
- Загрузка каталога (UI получает список товаров).
- Добавление в корзину (UI отправляет POST, получает обновлённую корзину).
- Загрузка отзывов (UI получает список и отображает).

### 2.2 Backend ↔ База данных
- Создание сущности (категория/товар).
- Чтение сущности по ID.
- Удаление сущности и проверка корректных ограничений.

### 2.3 Админ‑панель с БД
- Создание товара из админки.
- Изменение товара и проверка, что изменения сохранены.
- Удаление товара и проверка наличия в архиве/БД.

### 2.4 Поиск и фильтрация
- Фильтр по категории.
- Поиск по строке.
- Фильтр по цене.

## 3. Выполнение тестирования

Тест‑прогоны выполнены на локальном стенде с запущенным backend (`http://localhost:8080`).  
Для защищённых сценариев использовался админ JWT. Созданные тестовые сущности удалены после проверки.

## 4. Протокол интеграционных испытаний

| ID | Сценарий | Результат | Примечание |
| --- | --- | --- | --- |
| INT‑FE‑01 | Каталог: UI ↔ API `/products` | Pass | Получено 20 товаров |
| INT‑FE‑02 | Корзина: UI ↔ API `/cart/items` | Pass | Добавление товара увеличивает `totalItems` |
| INT‑BE‑01 | CRUD категория | Pass | Категория создаётся и читается по ID |
| INT‑ADM‑01 | Создание товара в админке | Pass | Товар создан, затем удалён |
| INT‑SRCH‑01 | Фильтр по категории | Pass | Возвращает товары только выбранной категории (в тесте 0) |

## 5. Несоответствия (баг‑репорт)

На момент составления документа дефекты не зафиксированы (прогоны выполнены, ошибок не выявлено).

## 6. Реальные пользовательские сценарии

1. Пользователь заходит в каталог → применяет фильтр → открывает карточку товара → добавляет в корзину.
2. Администратор создаёт товар → видит его в каталоге → удаляет (переходит в архив).
3. Пользователь оставляет отзыв → администратор одобряет → отзыв появляется на странице товара.

## 7. Итог

Документ подготовлен. Прогоны проведены, результаты внесены в раздел 4.
