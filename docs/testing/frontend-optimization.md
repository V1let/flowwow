# Заключение по frontend‑оптимизации

Дата: 20.02.2026  
Проект: Flowwow

## 1. Выполненные оптимизации CSS/JS

**Удаление неиспользуемого кода**
- Удалено подключение `data.js` на странице каталога (не использовался).
- Вынесены inline‑скрипты страниц восстановления пароля в отдельные файлы.

**Объединение и минимизация**
- Введена сборка фронтенда через `esbuild`.
- Для каждой страницы собирается единый JS‑бандл.
- CSS объединён в единый минифицированный файл.

**Изменённые пути:**
- CSS: `../dist/css/style.css?v=1`
- JS: `../dist/js/<page>.js?v=1`

## 2. Адаптивность

**Ручная проверка (без визуального прогона, по статической проверке CSS/HTML):**
- Страницы: `index`, `catalog`, `product`, `cart`, `favorites`, `admin`
- Брейкпоинты: 360×640, 768×1024, 1366×768

**Результаты:**
- `index` — OK (гриды на `auto-fit/auto-fill`, есть мобильные правила в `responsive.css`).
- `catalog` — OK (карточки и фильтры адаптируются сетками и медиаправилами).
- `product` — OK (колонки должны стекаться по мобильным правилам).
- `cart` — OK (табличная зона переходит в вертикальные блоки по медиаправилам).
- `favorites` — OK (сеточная раскладка карточек).
- `admin` — OK (таблицы и формы имеют моб. правила).

**Ограничение:** визуальный прогон в браузере не выполнялся (нет GUI в среде).
